Step 01 · continued

The prompt (part 2 of 2).

Paste this directly under part 1 in the same message. Send it once — Claude runs the rest of the setup interactively.

STEP 1 — THE AGENT FOLDER
Create a clean project folder (default: ~/content-agent). Inside it scaffold: CLAUDE.md (the agent's instructions — who I am, my niche, my voice, my competitors), a scripts/ folder, a dashboard/ folder, and a gitignored .env for tokens. Seed CLAUDE.md with my handle + competitors. Show me the folder tree before moving on.

STEP 2 — PULL THE REAL DATA
Write a small Node script that uses Apify's "instagram-scraper" (resultsType: posts, a high resultsLimit) to pull MY full post history + my competitors' recent posts, then ranks every post by views. Run it. Show me my true all-time top posts + my real follower count. (Important: do NOT use the "instagram-profile-scraper" latestPosts field — it only returns recent posts and will get my top post wrong.) Save the result to dashboard/data.json.

STEP 3 — BUILD THE DASHBOARD
Build a single self-contained dashboard (one index.html + a few small JS files) that shows: my key stats up top (followers, top post, total views, engagement), and a row of 5 "agent" cards — Ideator, Hook & Script, Planner, Analyst, DM Manager — each with a little working animation and a few live metrics. Make every section clickable so I can drill into what each agent produced. Use a clean, modern look (white background, one accent colour, soft shadows). Render it, screenshot it, and show me before we wire anything else.

STEP 4 — THE TELEGRAM BOT
Walk me through creating a Telegram bot: open Telegram, message @BotFather, send /newbot, name it, copy the token. Have me message my new bot once. Then write a small sender script that reads the token + my chat id from .env, pulls my chat id from getUpdates, and sends a digest (my real stats + top idea) to my phone. Run it and confirm it lands on my phone. Store the token in the gitignored .env — never print it.

STEP 5 — MAKE IT RUN ON ITS OWN
Now wire the loop: the agent re-pulls my data on a schedule (weekly is fine), refreshes the dashboard, and sends the digest to Telegram automatically. Use a cron job or launchd. Prove it by triggering one run end to end while I watch.

STEP 6 — PROVE THE WHOLE THING
Walk me through one full cycle: data pulls → dashboard updates → the 5 agents' output → the digest hits my phone. I want to watch it work, not take your word for it.

RULES
- One step at a time. Stop and wait for my "ok" before the next step.
- Verify everything against what's really on screen / on my phone — never assume.
- Keep tokens in a gitignored .env. Never print or commit them.
- Plain files I can read and edit. No black-box services.
- If something already exists (a folder, a bot), inspect it and improve it instead of overwriting.

Step 02 · Prep

Three things to have ready.

Claude will ask you for these on the first message. If you have them lined up before you paste the prompt, you shave 20 minutes off the build.

1 · Your Instagram handle

Just the handle — e.g. @karemops.ai. Your account must be public for the Apify scraper to read it. If it's private, flip it public for the pull, then flip it back.

2 · 3–5 competitor handles

People in your exact niche who make the content you want to make. Not the biggest accounts — the accounts closest to where you're going. Their top posts are the map.

3 · An Apify account + API token

Free tier is fine to start. Sign up at apify.com, go to Settings → API & Integrations → Personal API tokens, and copy the token. If you skip this, Claude will walk you through it — but pre-doing it saves a step.

Important: the prompt tells Claude to use instagram-scraper with resultsType: posts — not the profile scraper. This is the one mistake that makes your "top post" wrong. The prompt already handles it, but if you tweak the script later, keep this rule.

§ 03 · The Dashboard

Five agents. One screen.

This is what Claude builds in Step 3. A single index.html with your real stats up top and a row of 5 agent cards below. Every card is clickable — click any one and you drill into what that agent produced this week.

Agent 01

Ideator

Scouts topics from your niche + your top-performing posts + competitor wins. Returns a ranked list of ideas to film.

Agent 02

Hook & Script

Takes an idea, writes 6 hook variants, then a full short-form script in your voice. Beat-by-beat, retention-first.

Agent 03

Planner

Lays your posts on a calendar. What ships tomorrow, what ships Friday, what needs filming this weekend.

Agent 04

Analyst

Reads your data.json every week. Flags what's working, what died, and what to double down on. Your feedback loop.

Agent 05

DM Manager

Handles inbound. Categorises DMs into leads / fans / spam, drafts warm replies, surfaces the ones that need you personally. This is where the funnel lives.

Every card is clickable. The dashboard isn't just a status page — it's a control panel. Click Ideator, see this week's picks. Click Analyst, see the numbers. Click DM Manager, see the drafts.

Step 03 · Telegram

The digest lands on your phone.

A dashboard you have to open is a dashboard you forget. Telegram fixes that. Claude walks you through creating a bot in about two minutes.

The setup (Claude drives — you tap)

  1. Open Telegram. Search @BotFather. Message it.
  2. Send /newbot. Give your bot a name and a username.
  3. BotFather returns a token — a long string. Copy it.
  4. Message your new bot once (say "hi") so it can find your chat id.
  5. Paste the token to Claude. It writes it into a gitignored .env, pulls your chat id via getUpdates, and fires a test digest to your phone.

What lands on your phone

  • Your current follower count + delta from last week
  • Your top-performing post from the last pull, with a link
  • This week's top idea from the Ideator agent
  • Any DMs the DM Manager flagged as leads
Security note: the prompt is strict about this — tokens live in a gitignored .env, never printed, never committed. If Claude ever tries to echo your token back at you, tell it to stop. That rule is in the prompt for a reason.

Step 04 · Autopilot

Then hand it off to a cron job.

You built it once. Now wire the loop so it runs without you touching anything. Weekly is the sweet spot for content data — daily is overkill, monthly is stale.

The loop

  1. Cron fires the Apify pull. Fresh data lands in dashboard/data.json.
  2. The dashboard re-renders with the new numbers on next open.
  3. The Ideator + Analyst agents run against the new data.
  4. The Telegram sender ships the digest to your phone.

On macOS Claude will use launchd. On Linux, a plain crontab entry. Either way, you get to watch it fire end-to-end once before it goes autonomous.

The verification move

Don't take Claude's word for anything. When the schedule is wired, ask it to trigger one run manually while you watch. Data pulls → dashboard refreshes → digest lands. If any step fails silently, you catch it now, not in three weeks when you realise you haven't gotten a digest.

Rule from the prompt: "Verify everything against what's really on screen / on my phone — never assume." That's your job too. Make Claude prove every piece.
